# Структура проекта

```
upupa/
├── main.py            # точка входа: регистрация хэндлеров, запуск polling
├── config.py          # конфигурация: ключи, bot/dp/router, AI-клиенты (распил — этап 2)
├── prompts.py         # промпты и текстовые данные
├── core/              # инфраструктура
│   ├── middlewares.py
│   ├── upupa_utils.py
│   └── history_engine.py
├── features/          # функциональные блоки бота (настройки, статистика, фильтры)
├── services/          # внешние сервисы и обработка медиа (поиск, погода, ytp, мемы)
├── games/             # игры (крокодил, егра)
├── AI/                # AI-функции (talking, summarize, генерация картинок и т.д.)
└── tests/             # smoke-тесты импортов (см. CI: .github/workflows/tests.yml)
```

Правила:
- Файлы данных (json/txt/ogg) читаются относительно рабочей директории (корень репы).
- Секреты: `config_private.py` (локально, в .gitignore) или env-переменные.
- Каждый этап рефакторинга = отдельный коммит, после которого тесты зелёные.
